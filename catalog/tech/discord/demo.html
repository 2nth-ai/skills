Architecture

Interactions Endpoint — no WebSocket needed

The Discord Gateway requires a persistent WebSocket — Workers can't do that. The Interactions Endpoint is a pure HTTP push model. Discord delivers every event; you respond or defer.

Three steps to a live Discord bot

Developer portal, three secrets, one PUT to register commands. No manifests — commands are registered programmatically via the REST API.

Three secrets in the Worker. Then call the Discord REST API to register your commands — use guild commands during development (instant) and global commands for production.
# Three Worker secrets wrangler secret put DISCORD_PUBLIC_KEY wrangler secret put DISCORD_BOT_TOKEN wrangler secret put DISCORD_APP_ID # Register commands — guild (instant) or global (up to 1h) PUT https://discord.com/api/v10/applications/{APP_ID}/guilds/{GUILD_ID}/commands Authorization: Bot {BOT_TOKEN} Content-Type: application/json [{ "name": "ask", "description": "Ask the 2nth assistant", "options": [{ "name": "question", "type": 3, "required": true, "description": "Your question" }] }, { "name": "invoice", "description": "Check invoice status" }, { "name": "report", "description": "Generate a business report" }] → 200 [ { "id": "...", "name": "ask", "version": "..." }, ... ] ✓ /ask, /invoice, /report registered in guild

What the Discord bot changes

Without the bot
Discord is social — no structured way to query business data from a channel
Manual copy-paste of reports and alerts into Discord by humans
No slash commands — teams have to open the ERP or a browser to get data
Alerts require a separate monitoring tool and manual posting
AI tools live in other tabs — not where the team already communicates
Bot setup feels complex — Gateway, WebSockets, server hosting
With 2nth Discord bot
/ask, /invoice, /report bring ERP and AI into the channel where the team already is
Pub/Sub → Worker → Discord webhook posts overdue alerts, order events, and anomalies automatically
Rich embeds with action buttons — save to D1, mark as handled, escalate — without leaving Discord
Ephemeral replies (flags: 64) keep sensitive data private — only the user who asked sees it
Interactions Endpoint on Workers — no servers, no WebSocket, scales to zero
Cloud Scheduler → Worker → webhook posts scheduled reports to #reports every morning
