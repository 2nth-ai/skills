Gmail API

The Gmail REST API v1 gives programmatic access to a user's inbox — read, send, label, search, and watch for new messages. Combined with Cloudflare Workers and Workers AI, it's the foundation of AI-powered email automation.

Docs: https://developers.google.com/gmail/api/reference/rest

The 2nth Model: Gmail as an AI-powered ops layer

Base URL and auth

Base: https://gmail.googleapis.com/gmail/v1/users/{userId}/
userId: 'me' for the authenticated user, or user@domain.com for service account impersonation

All requests require Authorization: Bearer {access_token}. See tech/google for token management.

Minimum scopes:

OperationScope
Read messageshttps://www.googleapis.com/auth/gmail.readonly
Send emailhttps://www.googleapis.com/auth/gmail.send
Modify labelshttps://www.googleapis.com/auth/gmail.modify
Full accesshttps://mail.google.com/

Prefer gmail.readonly + gmail.send over gmail.modify or full access unless labels are required.

// Extract plain text body
function extractBody(payload: any): string {
  if (payload.mimeType === 'text/plain' && payload.body?.data) {
    return atob(payload.body.data.replace(/-/g, '+').replace(/_/g, '/'));
  }
  if (payload.parts) {
    for (const part of payload.parts) {
      const text = extractBody(part);
      if (text) return text;
    }
  }
  return '';
}
const body = extractBody(msg.payload);

Send a message

function buildEmail({ to, from, subject, html, replyToMessageId, threadId }: {
  to: string; from: string; subject: string; html: string;
  replyToMessageId?: string; threadId?: string;
}): string {
  const headers = [
    `To: ${to}`,
    `From: ${from}`,
    `Subject: ${subject}`,
    `MIME-Version: 1.0`,
    `Content-Type: text/html; charset=utf-8`,
    replyToMessageId ? `In-Reply-To: ${replyToMessageId}` : '',
    replyToMessageId ? `References: ${replyToMessageId}` : '',
  ].filter(Boolean).join('\r\n');

  const raw = `${headers}\r\n\r\n${html}`;
  return btoa(raw).replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');
}

await fetch('https://gmail.googleapis.com/gmail/v1/users/me/messages/send', {
  method: 'POST',
  headers: { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' },
  body: JSON.stringify({
    raw: buildEmail({ to, from, subject, html }),
    threadId, // include to keep in same thread
  }),
});

Real-time inbox: Watch API pipeline

The Watch API eliminates polling. New emails trigger a Pub/Sub notification → Cloudflare Worker processes within seconds.

Architecture

Gmail inbox (new email arrives)
  → Gmail pushes notification to Cloud Pub/Sub topic
  → Pub/Sub pushes to Cloudflare Worker endpoint
  → Worker fetches full message
  → Workers AI classifies intent
  → Claude drafts reply / extracts data / routes to D1

Step 1: Set up Pub/Sub topic

gcloud pubsub topics create gmail-watch
gcloud pubsub subscriptions create gmail-watch-sub \
  --topic=gmail-watch \
  --push-endpoint=https://your-worker.workers.dev/gmail/webhook \
  --ack-deadline=60

# Grant Gmail permission to publish to topic
gcloud pubsub topics add-iam-policy-binding gmail-watch \
  --member="serviceAccount:gmail-api-push@system.gserviceaccount.com" \
  --role="roles/pubsub.publisher"

Step 2: Register the watch

// POST to Gmail watch — expires after 7 days, must renew
async function registerWatch(token: string, projectId: string) {
  return fetch('https://gmail.googleapis.com/gmail/v1/users/me/watch', {
    method: 'POST',
    headers: { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' },
    body: JSON.stringify({
      topicName: `projects/${projectId}/topics/gmail-watch`,
      labelIds: ['INBOX'],
      labelFilterAction: 'include',
    }),
  }).then(r => r.json()); // returns { historyId, expiration }
}

Step 3: Handle the Pub/Sub push in a Worker

export default {
  async fetch(request: Request, env: Env): Promise<Response> {
    if (request.method !== 'POST') return new Response('Method not allowed', { status: 405 });

    const body = await request.json() as {
      message: { data: string; messageId: string; publishTime: string }
    };

    // Decode the Pub/Sub message
    const notification = JSON.parse(atob(body.message.data)) as {
      emailAddress: string;
      historyId: string;
    };

    // Fetch new messages since last historyId
    const token = await getAccessToken(env); // from tech/google
    const history = await fetch(
      `https://gmail.googleapis.com/gmail/v1/users/me/history?startHistoryId=${notification.historyId}&historyTypes=messageAdded`,
      { headers: { Authorization: `Bearer ${token}` } }
    ).then(r => r.json());

    for (const record of history.history ?? []) {
      for (const added of record.messagesAdded ?? []) {
        await processMessage(added.message.id, token, env);
      }
    }

    return new Response('OK', { status: 200 }); // ack to Pub/Sub
  }
};

async function processMessage(messageId: string, token: string, env: Env) {
  const msg = await fetch(
    `https://gmail.googleapis.com/gmail/v1/users/me/messages/${messageId}?format=full`,
    { headers: { Authorization: `Bearer ${token}` } }
  ).then(r => r.json());

  const body = extractBody(msg.payload);
  const subject = msg.payload.headers.find(h => h.name === 'Subject')?.value ?? '';
  const from = msg.payload.headers.find(h => h.name === 'From')?.value ?? '';

  // Classify with Workers AI
  const classification = await env.AI.run('@cf/meta/llama-3.1-8b-instruct', {
    messages: [
      { role: 'system', content: 'Classify this email as: enquiry, order, complaint, or spam. Reply with one word only.' },
      { role: 'user', content: `Subject: ${subject}\n\n${body.slice(0, 500)}` },
    ],
  });

  const intent = classification.response.trim().toLowerCase();

  // Store in D1
  await env.DB.prepare(
    'INSERT INTO emails (message_id, from_address, subject, intent, body, received_at) VALUES (?,?,?,?,?,?)'
  ).bind(messageId, from, subject, intent, body, Date.now()).run();

  // Apply label based on intent
  if (intent === 'enquiry' || intent === 'order') {
    await applyLabel(messageId, 'Needs Reply', token);
  }
}

Watch renewal (Cloud Scheduler)

// Endpoint: POST /gmail/renew-watch
// Triggered by Cloud Scheduler every 6 days
export async function renewWatch(env: Env) {
  const token = await getAccessToken(env);
  const result = await registerWatch(token, env.GOOGLE_PROJECT_ID);
  // Store new expiration in KV
  await env.KV.put('gmail_watch_expiry', result.expiration);
}

Gotchas

  • base64url encoding: Gmail body data uses base64url (- instead of +, _ instead of /) — always use replace(/-/g,'+').replace(/_/g,'/') before atob()
  • Thread vs message: threadId links messages in the same conversation — always pass threadId when replying to keep thread intact
  • Watch is per-user: One active watch per user. Calling watch again replaces the previous one — no need to stop() first
  • historyId gaps: If your Worker is down when messages arrive, use history.list with the last known historyId to catch up — don't rely on the Pub/Sub notification alone
  • MIME multipart: Most real emails are multipart/alternative — always recurse through parts to find text/plain or text/html
  • Label IDs not names: The modify endpoint requires label IDs (Label_12345) not names — always look up ID by name first and cache it
