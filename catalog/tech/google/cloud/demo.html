Connection

Three steps to GCP from Workers

A service account key stored as a Worker secret is all it takes. JWT minting happens in the V8 isolate — no Node.js, no external libraries.

GCP Connection Setup — 2nth-platform
Not connected
1
Create service account & download key
One service account per environment. Grant only the IAM roles the Worker actually needs — Cloud Run invoker, BigQuery data viewer, Pub/Sub publisher.
# Create the service account gcloud iam service-accounts create 2nth-worker-sa \ --display-name="2nth Worker → GCP" \ --project=2nth-platform # Grant Cloud Run invoker (call Cloud Run services) gcloud projects add-iam-policy-binding 2nth-platform \ --member="serviceAccount:2nth-worker-sa@2nth-platform.iam.gserviceaccount.com" \ --role="roles/run.invoker" # Grant BigQuery data viewer gcloud projects add-iam-policy-binding 2nth-platform \ --member="serviceAccount:2nth-worker-sa@2nth-platform.iam.gserviceaccount.com" \ --role="roles/bigquery.dataViewer" # Download the key (once only — store immediately) gcloud iam service-accounts keys create key.json \ --iam-account=2nth-worker-sa@2nth-platform.iam.gserviceaccount.com ✓ Created key [abc1234] for [2nth-worker-sa@2nth-platform.iam.gserviceaccount.com]
2
Store credentials as Worker secrets
The private key and client email go into Cloudflare as encrypted secrets. Never in wrangler.toml or source code. The Worker mints short-lived access tokens on demand using Web Crypto.
# Store each field from key.json as a separate secret wrangler secret put GCP_CLIENT_EMAIL # → 2nth-worker-sa@2nth-platform.iam.gserviceaccount.com wrangler secret put GCP_PRIVATE_KEY # → -----BEGIN RSA PRIVATE KEY----- (from key.json) wrangler secret put GCP_PROJECT_ID # → 2nth-platform # Mint a token in the Worker (Web Crypto, no Node.js required) async function getGCPToken(env: Env): Promise<string> { const now = Math.floor(Date.now() / 1000); const payload = { iss: env.GCP_CLIENT_EMAIL, scope: 'https://www.googleapis.com/auth/cloud-platform', aud: 'https://oauth2.googleapis.com/token', exp: now + 3600, iat: now }; // Sign with RSASSA-PKCS1-v1_5 / SHA-256 via crypto.subtle // → returns Bearer token valid for 1 hour }
3
First API call — invoke Cloud Run & query BigQuery
With a valid access token, the Worker can invoke Cloud Run services, run BigQuery queries, publish to Pub/Sub, or read from Cloud Storage — all via standard fetch().
# Test 1: List Cloud Run services in africa-south1 GET https://run.googleapis.com/v2/projects/2nth-platform/locations/africa-south1/services Authorization: Bearer {token} { "services": [ { "name": "erp-demo", "uri": "https://erp-demo-xxx-oa.a.run.app", "observedGeneration": 3, "latestReadyRevision": "erp-demo-00003-abc", "conditions": [{ "type": "Ready", "state": "CONDITION_SUCCEEDED" }] }, { "name": "frappe-staging", "uri": "https://frappe-staging-xxx-oa.a.run.app", "conditions": [{ "type": "Ready", "state": "CONDITION_SUCCEEDED" }] } ] } # Test 2: Run a BigQuery query POST https://bigquery.googleapis.com/bigquery/v2/projects/2nth-platform/queries { "query": "SELECT client, SUM(amount) as total FROM `2nth-platform.erp.invoices` WHERE date > DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY) GROUP BY client ORDER BY total DESC LIMIT 5", "useLegacySql": false } ✓ Connected — 2 Cloud Run services · 1 BigQuery dataset · africa-south1

What connecting GCP unlocks

The edge handles speed and intelligence. GCP handles scale and persistence. Together they cover every tier of the stack.

Without GCP
ERPNext only runs on a VPS — always on, always paying
Analytics require exporting CSVs and opening Excel
Background jobs block the main request thread
ML inference means a separate API key and external call for every request
Demo environments cost the same as production
Event pipelines need bespoke queue infrastructure
With Cloudflare + GCP
Cloud Run spins up ERPNext in ~2s, costs nothing at rest — demo on demand
BigQuery answers analytics SQL over millions of rows in seconds, from a Worker
Pub/Sub decouples order processing, invoicing, notifications — each scales independently
Vertex AI / Gemini on-demand for long-context document tasks Workers AI can't handle
Dev/staging on Cloud Run costs pennies — zero idle cost
Cloud Scheduler wakes services, renews tokens, triggers nightly syncs automatically
