Connection

Three steps to
WhatsApp Cloud API

Meta Business Manager, a System User token (non-expiring), and two Worker secrets. Webhook uses HMAC-SHA256 — same algorithm as Slack, different header name.

Create Meta app + add WhatsApp product
developers.facebook.com → Create App → Business → Add WhatsApp. Meta gives you a test number with 5 free contacts. Upgrade to a production number via a WhatsApp Business Service Provider or directly in Business Manager.
# developers.facebook.com → My Apps → Create App → Business # Add Product → WhatsApp → Set up App ID: 1234567890 App Secret: abc123def456... (Basic Settings) Phone Number ID: 987654321098765 (WhatsApp → Getting Started) WABA ID: 111222333444555 (WhatsApp Business Account ID) # Business Manager → System Users → Create system user # Add assets: WhatsApp Business Account → Manage # Generate token → select whatsapp_business_messaging scope System User Token: EAAxxxxxxx... # Non-expiring ✓ ✓ App created — test number active — system user token generated
2
Configure webhook + store secrets
Set the Callback URL to your Worker. Meta sends a GET to verify it — the Worker returns the hub.challenge. Then store the four secrets and deploy.
# WhatsApp → Configuration → Webhook Callback URL: https://your-worker.workers.dev/meta Verify Token: my-secret-verify-token-2026 # Subscribe to: messages # Meta verifies: GET /meta?hub.mode=subscribe&hub.challenge=xxx&hub.verify_token=... ← 200 "xxx" (raw challenge string, not JSON) ✓ # Store Worker secrets wrangler secret put META_APP_SECRET wrangler secret put WEBHOOK_VERIFY_TOKEN wrangler secret put WHATSAPP_TOKEN wrangler secret put WHATSAPP_PHONE_ID ✓ Webhook active — 4 secrets stored
3
Send test message + receive reply
Send a WhatsApp message to the test number. The webhook fires, the Worker classifies with Workers AI, Claude drafts the reply, and it's sent back via the Cloud API with an interactive button.

What WhatsApp AI
changes for a business

Without WhatsApp AI
Customers WhatsApp the owner's personal number — no separation, no scale
Invoice and order queries handled manually — owner stops what they're doing to reply
After-hours messages sit unanswered until the next morning
No record of what was asked or answered — context lost between conversations
Template messages sent manually by copying and pasting into chats one by one
WhatsApp Business app — no API, no automation, single device
With 2nth WhatsApp AI
Business number on the Cloud API — multiple agents, no device required
Invoice queries answered in 3 seconds — Workers AI classifies, D1 fetches data, Claude replies
24/7 first response — AI handles tier-1 immediately, escalates complex issues to a human
Full conversation history in D1 — context available on every message, CRM synced
Bulk template sends triggered by Pub/Sub or Cloud Scheduler — invoice reminders, order updates, proactive alerts
Interactive buttons guide customers to self-serve — fewer calls, faster resolution
