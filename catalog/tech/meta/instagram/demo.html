Architecture

🔐 Signature verify

Every POST is verified with HMAC-SHA256 against X-Hub-Signature-256 before processing. Same pattern as Meta/WhatsApp — reuse verifyMetaSignature().

⚡ Edge classification

Workers AI (Llama 3.1 8B) classifies intent at the edge — enquiry, order, collab, support, general — in ~50ms before calling Claude for the full reply.

🧵 Conversation context

Last 6 turns stored in KV keyed by IGSID with 7-day TTL, matching the session window. Claude gets full context for coherent multi-turn conversations.

📸 Story mentions

Story mention events arrive via changes[field="mentions"], not messaging[]. Fetch story metadata immediately — story URLs expire after 24 hours.

🤝 Handover protocol

When Claude detects an escalation phrase, control is passed to the human agent app via /pass_thread_control. AI resumes automatically after resolution.

🗃️ D1 persistence

Message ID (mid) stored with UNIQUE constraint for deduplication. Separate table for story mentions. Both indexed on IGSID for CRM lookups.

Link Instagram Professional Account

Instagram Messaging requires a Business or Creator account connected to a Facebook Page in Meta Business Manager.

Go to business.facebook.com → Accounts → Instagram Accounts
Connect your Instagram Professional Account to a Facebook Page
In Meta App Dashboard → Add Product → Messenger (Instagram uses the Messenger Platform)
Under Instagram → Settings, enable "Access to Messages"
Request permissions: instagram_manage_messages, pages_messaging

Configure the webhook

Subscribe to Instagram messaging events in the App Dashboard. The verify token can be any string you choose.

# App Dashboard → Webhooks → Add Subscription # Object: instagram # Fields to subscribe: messages # DMs messaging_postbacks # button taps messaging_story_mentions # story mentions message_reads # optional read receipts
Callback URL: https://your-worker.workers.dev/webhook
Verify token: any string — stored as WEBHOOK_VERIFY_TOKEN secret
Worker handles GET verification by returning raw hub.challenge string

Add secrets and deploy

Four secrets needed. Use a long-lived System User token in production — Page tokens expire when the user changes password.

wrangler secret put META_APP_SECRET # App Secret from developers.facebook.com → Basic Settings wrangler secret put WEBHOOK_VERIFY_TOKEN # Must match what you entered in App Dashboard wrangler secret put PAGE_ACCESS_TOKEN # Long-lived token with instagram_manage_messages wrangler secret put INSTAGRAM_PAGE_ID # Numeric Page ID (not @handle)
Deploy: wrangler deploy — Worker handles GET verify + POST events
Send yourself a test DM — you should get an AI reply within ~4 seconds
Check D1: wrangler d1 execute DB --command "SELECT * FROM ig_messages"
