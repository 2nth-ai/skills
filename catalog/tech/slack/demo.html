Architecture

The 3-second rule and how to beat it

Slack kills any endpoint that doesn't respond in 3 seconds. AI takes 2–10 seconds. The pattern: ack instantly, process with ctx.waitUntil(), post the reply via Web API when ready.

Connection

Three steps to a live Slack bot

Create the app, store two secrets, wire the Worker URL. No servers, no SDK, no OAuth dance for bot tokens.

Slack bot setup — 2nth-workspace
Not installed
1
Create Slack app from manifest
api.slack.com/apps → Create New App → From an app manifest. Paste the YAML — it configures scopes, event subscriptions, and slash commands in one step.
# slack-app-manifest.yaml display_information: name: 2nth Assistant background_color: "#4A154B" features: bot_user: { display_name: 2nth, always_online: true } slash_commands: - command: /ask url: https://your-worker.workers.dev/slack/command oauth_config: scopes: bot: [ app_mentions:read, chat:write, commands, channels:history, users:read ] settings: event_subscriptions: request_url: https://your-worker.workers.dev/slack/events bot_events: [ app_mention, message.im ] interactivity: request_url: https://your-worker.workers.dev/slack/interact ✓ App created — Install to workspace to get the bot token
2
Store bot token + signing secret
After installing the app, copy the Bot Token from OAuth & Permissions and the Signing Secret from Basic Information. These are the only two credentials the Worker needs.
# Store as Worker secrets — never in wrangler.toml wrangler secret put SLACK_BOT_TOKEN # → xoxb-YOUR-BOT-TOKEN-HERE wrangler secret put SLACK_SIGNING_SECRET # → 8f742231b10e8888abcd99baed5f6149 # Signature verification in the Worker (HMAC-SHA256) const sigBase = `v0:${timestamp}:${body}`; const key = await crypto.subtle.importKey( 'raw', encoder.encode(env.SLACK_SIGNING_SECRET), { name: 'HMAC', hash: 'SHA-256' }, false, ['sign'] ); const hex = 'v0=' + toHex(await crypto.subtle.sign('HMAC', key, encoder.encode(sigBase))); // hex === X-Slack-Signature header → verified ✓
3
Verify URL + first @mention
Slack sends a challenge to the events URL on setup. The Worker responds with the challenge value. Then send a test @mention — the bot should reply in thread within a few seconds.
# Slack sends challenge on URL verification POST /slack/events { "type": "url_verification", "challenge": "3eZbrw1oBIq..." } → 200 { "challenge": "3eZbrw1oBIq..." } # First app_mention event POST /slack/events { "event": { "type": "app_mention", "user": "U0LAN0Z89", "text": "<@U061F7AUR> show overdue invoices", "channel": "C024BE91L", "ts": "1743500000.123456" } } # Worker acks instantly (<100ms) → 200 OK (empty body) # ctx.waitUntil() → Workers AI → Claude → chat.postMessage ✓ Bot active — @2nth responding in #general · thread replies enabled

Before & After

What adding an AI bot changes

Without the bot
Team switches context to check the ERP for every stock or invoice question
Managers chase updates in threads manually — no structured data, just chat
Slash command ideas require a full backend build to implement
Alerts and notifications require separate monitoring tools and manual copy-paste into Slack
AI tools are separate tabs — ChatGPT, Claude.ai — not where the work is happening
Report requests become calendar invites and spreadsheet emails
With 2nth Slack bot
@2nth answers ERP, invoice, and stock queries in the channel where the conversation already is
Structured Block Kit replies with action buttons — save to D1, open in ERP, send to client
/ask spins up a slash command instantly — Workers AI classifies, Claude drafts, zero infrastructure
GCP Pub/Sub → Worker → Incoming Webhook sends overdue alerts automatically to #ops-ai
AI is in Slack — the tool the team already uses all day, not another tab to switch to
Reports drafted and posted on schedule via Cloud Scheduler → Worker → chat.postMessage
