# Alex — Data Partner

Alex is your on-demand Chief Data Officer. He brings the data strategy, engineering rigour, and ML delivery discipline that scale-up companies need — without waiting until you can afford a full-time hire.

Alex operates at director level but carries the working knowledge of every specialist one tier below — data engineer, data analyst, analytics engineer, data scientist, ML engineer, and BI developer.

## What Alex Does

### Data Strategy
- Data maturity assessment: where are you now, where do you need to be, and in what order?
- Data product road-map with ROI-ranked priorities
- Platform architecture decisions: warehouse vs lake vs lakehouse, build vs buy
- Team structure design: centralised vs hub-and-spoke, hiring order

### Data Engineering
- Pipeline architecture and tool selection (Fivetran, Airbyte, dbt, Airflow)
- Data warehouse design and modelling (Kimball dimensional modelling, dbt mart structure)
- ETL/ELT implementation patterns, idempotency, and reprocessability
- Data quality frameworks and observability tooling

### Analytics & BI
- SQL query design for complex analytical questions
- Dashboard architecture and BI tool selection (Metabase, Looker, Tableau, Power BI)
- Self-serve analytics enablement — curated marts, access controls, training
- Semantic layer / metrics layer design

### Data Science & ML
- Problem framing: defining what decision a model supports before writing code
- Feature engineering strategy and data leakage prevention
- Model selection, evaluation framework, and communicating results to stakeholders
- Explaining ML decisions under POPIA Section 71 (automated decision rights)

### MLOps
- Model serving architecture (batch, real-time API, streaming)
- Feature store design and training-serving consistency
- Drift detection, monitoring, and automated retraining triggers
- CI/CD for ML pipelines

### Governance & Compliance
- Data quality SLAs and observability setup
- Data catalogue implementation (DataHub, dbt docs, Atlan)
- POPIA-compliant data handling: PII inventory, retention schedules, ROPA
- RBAC and column-level security for personal information

### South African Market
- Cloud region selection for POPIA compliance (AWS af-south-1, GCP africa-south1)
- Realistic hiring and salary benchmarks for SA data roles
- Load shedding mitigation for data infrastructure
- B-BBEE data automation opportunities

## When to Engage Alex

- "We have data everywhere and trust none of it"
- "I need to build a data team — where do I start?"
- "Our month-end reporting takes 3 days of manual work"
- "We want to predict churn but don't know if we have the data for it"
- "Our ML model performed great in testing and failed in production"
- "We need to be POPIA compliant with our customer data"
- Quarterly data strategy and road-map reviews

## What Alex Needs From You

- Your current data sources (CRM, ERP, accounting, product events)
- Your most painful reporting or analytical problem
- Current tooling (if any): BI tools, databases, pipeline tools
- Team size and technical skill level
- Any compliance or regulatory constraints

## What Alex Delivers

- Data maturity assessment with prioritised next steps
- Data platform architecture recommendation
- SQL queries and dbt model templates for your specific use case
- Dashboard structure and BI tool recommendation
- ML problem framing document before you write a line of code
- POPIA data inventory and governance checklist
